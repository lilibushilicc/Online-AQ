# 项目整体说明

## 1. 项目定位

`Online-AQ` 是一个面向教学与练习场景的智能在线答题系统，采用前后端分离架构，并同时提供：

- **Web 管理端**：面向教师/管理员，负责题库、考试、成绩、反馈、配置等后台管理。
- **Web 学生端**：面向学生，负责考试作答、成绩复盘、在线练习、错题整理、个人中心等学习闭环。
- **Android 学生端 App**：面向手机端学生用户，承载移动场景下的考试、成绩、练习、错题本和个人中心。

项目的核心目标不是单点做题，而是提供一整套从“题目录入 -> 组卷发布 -> 学生作答 -> 自动评分 / 人工评分 -> 成绩复盘 -> 错题沉淀 -> 反馈修正”的闭环流程。

## 2. 项目组成

### 2.1 Web 前端

目录：`Oline-AQ-vue/`

技术栈：

- Vue 3
- Vite
- TypeScript
- Element Plus
- Vant
- Pinia
- Axios

作用：

- 管理员在网页端管理系统后台。
- 学生在网页端完成考试、练习、成绩查看、错题本管理、个人资料维护。
- 同一套学生业务页支持桌面网页和手机网页访问。

### 2.2 Spring 后端

目录：`Oline-AQ-spring/`

技术栈：

- Spring Boot
- MyBatis-Plus
- JWT
- PostgreSQL
- Spring Cache + Caffeine

作用：

- 提供认证、题库、考试、成绩、错题本、公告、反馈、系统配置等全部接口。
- 管理考试发布、草稿恢复、重考、稳定乱序映射、评分和导出能力。
- 统一处理角色鉴权、业务规则和数据库交互。

### 2.3 Android 学生端

目录：`Oline-AQ-app/`

技术栈：

- Kotlin
- Android View + Material 3
- Retrofit

作用：

- 为学生提供独立的手机端使用入口。
- 覆盖考试、成绩、练习、错题本、个人中心、公告等移动场景。

## 3. 主要角色

### 3.1 管理员 / 教师

管理员主要负责：

- 上传并解析题库文件
- 维护题目内容、分类和分值
- 创建考试、发布考试、设置考试规则
- 查看学生成绩、处理简答题评分
- 处理题目纠错反馈
- 配置系统参数、SMTP、AI 接口、对象存储等

### 3.2 学生

学生主要使用能力包括：

- 查看可参加考试并在线作答
- 自动保存草稿并恢复作答
- 查看成绩详情与历次提交记录
- 进行在线练习
- 管理错题本
- 查看公告、修改资料、提交题目反馈

## 4. 核心业务模块

### 4.1 认证与权限

- 登录基于 JWT。
- 后端通过拦截器解析 Token。
- 管理员接口通过统一的 `@AdminOnly` 注解控制访问权限。
- 学生和管理员共用一套后端，但访问范围不同。

### 4.2 题库模块

- 支持手动新增题目。
- 支持上传 TXT / DOCX 文件解析入库。
- 支持分类、批量设分、批量设分类、导出题库。
- 题目类型覆盖单选、判断、填空、简答。

### 4.3 考试模块

- 管理员创建考试草稿并选择题目。
- 支持发布为全体学生考试或指定学生考试。
- 支持考试开始/结束时间窗口。
- 支持题目乱序、选项乱序。
- 支持学生服务端草稿恢复。
- 支持正式重考，作答按 `attemptId` 会话管理。
- 支持稳定乱序映射，确保刷新或跨设备恢复后题序一致。

### 4.4 成绩与评分模块

- 客观题自动评分。
- 简答题支持教师人工评分。
- 学生成绩按考试聚合，也支持查看历次提交明细。
- 成绩详情、错题本和导出链路已收敛为明确 VO，字段边界更清晰。

### 4.5 练习模块

- 学生可从题库中按分类进行练习。
- 支持本地判分与参考答案展示。
- 已支持练习结果持久化与历史记录查看。

### 4.6 错题本模块

- 学生可查看全部错题。
- 支持创建多个错题本。
- 支持添加、移除、编辑、删除错题本内容。
- 错题按考试维度分组，便于回溯来源。

### 4.7 反馈与公告模块

- 学生可对题目提交纠错反馈。
- 管理员可查看反馈、采纳并修改题目，或驳回反馈。
- 系统公告支持创建、查看、已读标记。
- Web 学生端支持 SSE 实时通知。

### 4.8 系统配置模块

- 支持对象存储、AI 接口、SMTP、邮件发送限制等配置。
- 支持测试 R2、测试邮箱、测试 AI 接口连接。

## 5. 三端关系说明

### 5.1 网页端桌面与网页端手机

- 两者共用同一套学生业务路由和接口。
- 手机网页端通过 `StudentMobileLayout` 提供移动壳布局。
- 功能结果基本一致，但部分页面仍是桌面业务页复用，移动体验尚未完全专门化。

### 5.2 网页端与 Android App

- 核心学生主流程已经大体对齐。
- Android App 可以视为学生侧独立移动产品形态。
- 但仍有少数网页端细节能力需要继续核实是否在 App 中完全覆盖，例如导出、服务端草稿恢复、练习历史等。

## 6. 当前项目状态概括

当前项目已经不是单一原型，而是一个具备较完整闭环的教学答题系统：

- 管理端功能较完整。
- 学生网页端功能较完整。
- Android 学生端主流程较完整。
- 后端已经补齐统一管理员鉴权、考试重考与稳定乱序映射、结果链路类型收敛等基础能力。

当前更值得继续投入的方向主要有三类：

1. **移动端深度体验优化**
说明：网页手机端仍有部分页面偏桌面化。

2. **三端功能继续核对补齐**
说明：重点确认 Android 端少数未完全明确项。

3. **继续提升结构清晰度**
说明：项目中仍有部分接口继续使用 `Map<String, Object>`，后续可按优先级逐步收敛为明确 VO。
