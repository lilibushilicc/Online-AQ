# 考试模块

## 1. 模块概览

考试模块负责管理员组卷与发布、学生拉取考试详情、草稿暂存、正式提交、重考记录沉淀以及成绩明细回放。

当前实现支持以下关键能力：

- 管理员创建考试草稿并设置题目分值。
- 发布考试时可指定全体学生或指定学生。
- 可选开启题目乱序、选项乱序。
- 学生端支持服务端草稿恢复。
- 学生作答中支持退出考试确认，离开页面前提示并保留当前草稿。
- 学生可在题目未全部完成时主动提交，未作答题按空答案处理，并在提交前收到明确确认提示。
- 支持正式重考，多次提交分别形成独立成绩记录。
- 每次作答使用独立 `attemptId` 标识，并绑定稳定乱序映射，保证刷新和跨设备恢复后题目顺序一致。
- 倒计时结束后自动交卷，自动交卷不受退出确认弹框影响。
- 学生端网页与 Android 历史详情都可查看单次提交对应的题目、选项、你的答案与正确答案信息。

## 2. 核心数据结构

### 2.1 exam

- `exam_id`：考试 ID
- `exam_name`：考试名称
- `duration`：考试时长（分钟）
- `status`：`draft / published / closed`
- `allow_retake`：是否允许重考
- `assign_all`：是否分配给全体学生
- `shuffle_questions`：是否乱序题目
- `shuffle_answers`：是否乱序选项
- `start_time / end_time`：开放时间窗口

### 2.2 draft_answer

- `exam_id + student_id`：每个学生在每场考试只保留一份当前草稿
- `attempt_id`：当前作答会话 ID
- `answers`：草稿答案 JSON
- `shuffle_snapshot`：当前作答会话的稳定乱序快照 JSON
- `use_time`：当前作答已用秒数

### 2.3 exam_result

- `result_id`：单次提交成绩 ID
- `exam_id`：考试 ID
- `student_id`：学生 ID
- `attempt_id`：本次提交对应的作答会话 ID
- `submit_time`：提交时间

### 2.4 student_answer

- `answer_id`：答题明细 ID
- `attempt_id`：所属作答会话 ID
- `question_id`：题目 ID
- `student_answer`：学生作答
- `correct_answer`：标准答案
- `score`：该题得分

## 3. 作答与重考流程

### 3.1 首次进入考试

1. 学生请求 `GET /api/exams/{examId}`。
2. 服务端校验考试是否已发布、学生是否被分配到该考试。
3. 若该学生在 `draft_answer` 中还没有记录，则创建一条草稿并生成新的 `attemptId`。
4. 若考试开启题目乱序或选项乱序，则首次进入时生成稳定乱序快照并保存到 `shuffle_snapshot`。
5. 返回考试详情、题目列表、`attemptId` 和当前乱序映射。

### 3.2 草稿保存与恢复

1. 学生端把当前 `attemptId` 连同答案和已用时间一起提交到 `PUT /api/exams/{examId}/draft`。
2. 服务端按 `exam_id + student_id` 更新当前草稿，不更换 `attemptId`。
3. 学生刷新页面、换设备重新进入时，先请求考试详情，再通过 `GET /api/exams/{examId}/draft` 恢复同一 `attemptId` 下的答案。
4. 因为乱序快照保存在服务端，恢复后的题目顺序和选项顺序与上次一致。

### 3.3 正式提交

1. 学生端调用 `POST /api/exams/{examId}/submit`，提交当前 `attemptId`、答案和用时。
2. 服务端基于草稿中的稳定乱序快照，把页面上的选项字母还原到原始答案语义后再评分。
3. 即使存在未作答题，前端也允许提交；服务端会把这些题按空答案处理并参与结果统计。
4. 生成一条 `exam_result` 和多条 `student_answer`，都绑定当前 `attemptId`。
5. 提交成功后，服务端清空当前草稿答案和乱序快照，并生成新的 `attemptId`，为下一次重考做准备。
6. 系统启动时会自动清理历史遗留的 `exam_result(exam_id, student_id)` 唯一约束/唯一索引，避免允许重考的考试仍被数据库层错误拦截。

### 3.4 退出考试与自动交卷

1. 学生开始作答后，页面右侧提供“退出考试”按钮。
2. 点击退出、点击站内其他菜单、使用浏览器后退离开考试页时，前端会弹出确认框。
3. 学生确认退出后，前端先保留当前作答草稿，再返回考试列表；下次重新进入会恢复草稿继续作答。
4. 学生取消退出时，继续停留在当前考试页作答。
5. 浏览器刷新或关闭标签页时，浏览器会显示原生离开提示，前端同时把当前答案缓存到本地草稿。
6. 当剩余时间归零时，前端直接触发自动交卷，并提示“考试时间已到，系统已自动交卷”；此流程不会弹出退出确认框，也不会被离开拦截阻断成绩页跳转。

### 3.5 重考

当 `allow_retake=true` 时：

- 同一学生可多次提交同一考试。
- 每次提交都会生成新的 `result_id` 和新的 `attemptId`。
- 成绩页按 `examId` 聚合，历史页展示每次提交记录。

## 4. 接口说明

| 接口 | 方法 | 作用 | 关键参数 | 说明 |
|---|---|---|---|---|
| `/api/exams/student` | GET | 获取学生可参加考试列表 | 无 | 仅返回被分配到的考试 |
| `/api/exams/{examId}` | GET | 获取考试详情 | `attemptId` 可选 | 学生首次进入可不传，服务端自动生成 |
| `/api/exams/{examId}/draft` | PUT | 保存草稿 | `attemptId`、`answers`、`useTime` | 绑定当前作答会话 |
| `/api/exams/{examId}/draft` | GET | 读取草稿 | 无 | 返回当前草稿与 `attemptId` |
| `/api/exams/{examId}/draft` | DELETE | 清除草稿 | 无 | 多用于提交后或手动清空 |
| `/api/exams/{examId}/submit` | POST | 提交试卷 | `attemptId`、`answers`、`useTime` | 服务端按稳定乱序映射评分 |
| `/api/results/my` | GET | 获取我的成绩 | 无 | 包含重考后的多条成绩记录 |
| `/api/results/{resultId}` | GET | 查看单次成绩详情 | `resultId` | 通过 `attemptId` 回放对应明细 |

## 5. 当前约束

- 学生只能访问已发布且分配给自己的考试。
- 重考能力依赖 `attemptId`，前后端不得再使用“同一考试只能有一条成绩”的旧假设。
- 如果数据库来自旧版本，必须移除 `exam_result(exam_id, student_id)` 这类旧唯一约束，否则会出现前端看似可重考、提交时却被服务端判成重复提交的问题。
- 题目乱序与草稿恢复必须同时开启同一份会话快照，不能每次详情请求重新随机。

## 6. 验证建议

```sh
cd Oline-AQ-spring
.\mvnw.cmd -DskipTests compile

cd ..\Oline-AQ-vue
npm run build
```

手工验证建议：

1. 开启允许重考，连续提交两次，确认成绩页出现两条历史记录。
2. 开启题目乱序与选项乱序，作答后刷新页面，确认题号顺序和选项顺序不变。
3. 切换设备或浏览器后登录同一账号，确认能恢复同一会话草稿。
4. 作答中点击“退出考试”或切换到其他学生端菜单，确认会弹出退出提示，确认退出后再次进入可恢复草稿。
5. 将考试剩余时间压缩到 1 分钟内，确认倒计时结束后会自动交卷并跳转到成绩结果。
6. 保留部分题目未作答直接提交，确认 Web 与 Android 都会先提示剩余未作答数量，确认后可成功交卷。
7. 在 Android 成绩列表进入某次历史提交详情，确认页面能显示题干、题目选项、你的答案与正确答案。
