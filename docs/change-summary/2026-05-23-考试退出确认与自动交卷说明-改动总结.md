# 考试退出确认与自动交卷说明改动总结

## 1. 改动目标

学生在考试作答过程中如果误点菜单、浏览器后退或主动退出，原实现不会给出明确确认，容易造成作答中断。此次改动为学生考试页补充退出确认弹框与离开拦截，同时保持倒计时到期后的自动交卷能力不变。

## 2. 用户视角

学生进入考试并开始作答后，右侧操作区会新增“退出考试”按钮。

学生点击“退出考试”、点击站内其他菜单切换页面或使用浏览器后退离开考试页时，系统会弹出确认框，提示当前作答草稿会被保留。

学生确认退出后会返回考试列表，下次重新进入同一考试时可恢复上次草稿继续答题；取消退出则继续留在当前页面作答。

如果考试时间到达，系统仍会自动交卷并提示“考试时间已到，系统已自动交卷”，不需要学生手动确认退出；交卷成功后的成绩页跳转不会再被退出拦截误伤。

本次联调还发现一个隐藏问题：部分旧库仍残留 `exam_result(exam_id, student_id)` 唯一约束或唯一索引，导致允许重考的考试在提交时被数据库层误判为重复提交。现在后端启动时会自动清理这些历史遗留约束，并在草稿会话缺失时补齐 `attemptId`。

## 3. 前端实现

- 页面：`StudentExamDetail.vue` 在答题侧栏新增“退出考试”按钮。
- 组件：复用 Element Plus 的 `ElMessageBox.confirm` 作为退出确认弹框。
- 路由：使用 `onBeforeRouteLeave` 拦截站内路由离开，统一走退出确认逻辑；提交成功后显式放行成绩页跳转，避免误触发退出确认。
- 状态管理：通过 `started`、`attemptId`、`submitting` 与 `leavingConfirmed` 判断当前是否需要拦截离开以及避免重复弹框。
- 接口请求：退出确认后仍复用现有 `saveDraftApi` 保存服务端草稿，不新增接口。
- 样式与布局：退出按钮放置在提交试卷按钮下方，保持现有页面布局风格一致。

## 4. 后端实现

- Controller：无改动，继续复用现有草稿保存与正式交卷接口。
- Service：`ExamService.getOrCreateDraft()` 增加空 `attemptId` 兜底补齐；`SchemaInitializer` 启动时自动清理 `exam_result(exam_id, student_id)` 的旧唯一约束/唯一索引。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

## 5. 数据库与数据流

学生作答时，答案仍按原有链路定时保存到 `draft_answer` 草稿表。

当学生主动退出考试时，前端先调用草稿保存接口，再把同一份答案缓存到浏览器本地存储；重新进入考试页后，系统会优先通过草稿接口恢复服务端草稿，并在需要时兜底读取本地草稿。

当倒计时归零时，前端直接调用正式交卷接口，服务端生成 `exam_result` 与 `student_answer` 数据，并清理当前草稿会话。

如果数据库来自旧版本，系统在启动阶段会先删除 `exam_result(exam_id, student_id)` 的历史唯一约束/唯一索引，避免新版本的多次提交能力被旧库结构拦截。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/{examId}/draft` | PUT | 退出前或作答中保存草稿 | `attemptId`、`answers`、`useTime` | 无 |
| `/api/exams/{examId}/draft` | GET | 重新进入考试时恢复草稿 | 无 | 草稿内容、`attemptId` |
| `/api/exams/{examId}/submit` | POST | 倒计时结束自动交卷或手动交卷 | `attemptId`、`answers`、`useTime` | `resultId`、得分统计 |

## 8. 验证方式

```sh
cd Oline-AQ-spring
.\mvnw.cmd -DskipTests compile

cd Oline-AQ-vue
npm run build
```

手工验证重点：

1. 进入考试并开始作答后点击“退出考试”，确认会弹出提示。
2. 在确认退出后重新进入同一考试，确认草稿能恢复。
3. 在考试过程中点击学生端其他菜单或使用浏览器后退，确认同样会弹出退出提示。
4. 倒计时归零后确认会自动交卷并跳转成绩页。
5. 使用学生账号实际调用考试详情与提交接口，确认返回值包含 `attemptId`，且提交不再报“该考试已提交过，不允许重复提交”。

## 9. 风险与后续优化

- 浏览器刷新或关闭标签页场景只能使用浏览器原生离开提示，无法展示 Element Plus 自定义弹框，这是浏览器安全限制。
- 当前离开前的本地缓存为兜底能力，若需要在关闭标签页瞬间也稳定落服务端草稿，可后续评估 `sendBeacon` 或后端补充轻量草稿上报接口。
- 若后续希望支持“退出时直接交卷”与“保存草稿退出”双动作，可在当前确认框基础上扩展为自定义对话框。
