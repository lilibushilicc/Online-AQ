# 答案返回策略收口改动总结

## 1. 改动目标

在前两轮安全修复后，学生端考试详情未提交前的答案泄露已经被封住，后台管理接口越权也已经补上管理员校验。

这一轮继续排查成绩详情、错题本、导出链路，目标不是一刀切隐藏所有正确答案，而是明确哪些场景属于“允许的提交后复盘能力”，哪些场景属于“不应默认暴露答案的返回结构”，并把代码层的答案返回策略收紧为显式控制。

## 2. 用户视角

- 学生在正式提交后，仍然可以在成绩详情、错题本、导出文件中看到自己的正确答案，用于复盘。
- 学生在未提交前，仍无法通过考试详情接口直接拿到标准答案。
- 管理员原有评分、成绩查看、反馈处理等使用方式不变。

## 3. 前端实现

- 页面：无改动。
- 组件：无改动。
- 路由：无改动。
- 状态管理：无改动。
- 接口请求：无改动。
- 样式与布局：无改动。

## 4. 后端实现

- `AnswerMapHelper.toAnswerMap()` 改为必须显式传入 `includeCorrectAnswer`，不再默认总是返回 `correctAnswer`。
- `ResultService` 在成绩详情、错题本、导出相关链路中明确传入 `includeCorrectAnswer=true`，表示这些接口属于“当前产品允许的提交后复盘场景”。
- 本次没有新增新的学生公开接口，而是把未来的答案返回风险点前移到代码调用层，避免后续新增接口时因为复用工具方法而默认暴露答案。

## 5. 数据库与数据流

数据库无变化。

数据流上，本次确认并固化了两类策略：

1. **未提交前场景**：考试详情接口返回脱敏题目，不含正确答案。
2. **已提交后场景**：成绩详情、错题本、导出链路可返回正确答案，但前提是已经通过“本人或管理员”权限校验。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/{examId}` | GET | 获取考试详情 | `examId` | 学生端不返回 `correctAnswer` |
| `/api/results/{resultId}` | GET | 获取本人或管理员可见的成绩详情 | `resultId` | 保留正确答案用于复盘 |
| `/api/results/export/{resultId}/detail` | GET | 导出成绩详情 Excel | `resultId` | 保留正确答案用于复盘 |
| `/api/results/export/wrong` | GET | 导出学生自己的错题 | 无 | 保留正确答案用于复盘 |
| `/api/wrong-notebooks/{notebookId}` | GET | 查看错题本详情 | `notebookId` | 保留正确答案用于复盘 |

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-spring/src/main/java/com/example/olineaqspring/utils/AnswerMapHelper.java` | 答题明细返回结构改为显式控制是否包含正确答案 |
| `Oline-AQ-spring/src/main/java/com/example/olineaqspring/service/ResultService.java` | 明确哪些提交后复盘链路允许返回正确答案 |

## 8. 验证方式

```sh
cd Oline-AQ-spring
.\mvnw.cmd -DskipTests compile
```

验证结果：

- 后端编译通过。

补充审计结论：

- `考试详情`：学生端已脱敏，不再泄露答案。
- `成绩详情 / 错题本 / 相关导出`：答案展示是当前产品允许的“提交后复盘”功能，不属于未授权旁路泄露。

## 9. 风险与后续优化

- 当前答案返回策略已经显式化，但仍建议后续补一层统一 VO，而不是直接返回 `Map<String, Object>`，让不同场景的字段边界更清晰。
- 若后续产品希望支持“提交后也不立即展示正确答案”，可以在 `ResultService` 层继续基于考试配置增加条件控制，而不需要再大改工具链路。
