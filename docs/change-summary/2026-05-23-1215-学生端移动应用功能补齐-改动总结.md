# 学生端移动应用功能补齐改动总结

## 1. 改动目标

本次改动针对 `Oline-AQ-app` Android 学生端应用，目的是把当前 App 里缺失的学生功能补齐到更接近网页端的水平。

改动前，App 仅覆盖考试列表、考试作答、成绩列表、成绩详情和更多设置，学生端缺少在线练习、错题本、个人中心、系统公告查看、题目纠错反馈等网页端已有功能，导致手机端使用闭环不完整。

## 2. 用户视角

- 学生打开 App 后，底部导航从原来的 3 个入口扩展为 5 个入口：考试、成绩、练习、错题、我的。
- 学生可以直接在手机上做题库练习，并在提交后查看参考答案。
- 学生可以查看全部错题，创建自己的错题本，把错题加入错题本，或从错题本中移除。
- 学生可以在成绩详情中对题目发起纠错反馈。
- 学生可以在个人中心查看资料、修改资料、修改密码，并查看系统公告。

## 3. 前端实现

说明：这里的“前端”指 Android App 客户端界面与交互实现。

- 页面：
  - 新增 `PracticeFragment` 在线练习页。
  - 新增 `WrongBookFragment` 错题本首页。
  - 新增 `WrongBookDetailActivity` 错题本详情页。
  - 新增 `ProfileFragment` 个人中心页。
  - 扩展 `ResultDetailActivity`，增加题目反馈能力。
- 组件：
  - 新增多个布局文件用于练习题卡、错题本卡片、错题分组、公告卡片、资料编辑弹窗等。
  - 底部导航增加 5 个入口并补充对应图标资源。
- 路由：
  - Android 端通过 `MainActivity` + Fragment 切换实现页面导航。
  - `WrongBookDetailActivity` 作为独立详情页加入 `AndroidManifest.xml`。
- 状态管理：
  - 继续复用现有 `TokenManager`、`ThemeConfig`、`ServerConfig`。
  - 新增模型和接口响应对象承接网页端已有学生功能接口。
- 接口请求：
  - Retrofit `ApiService` 新增题库、分类、错题本、个人资料、公告、反馈相关接口。
- 样式与布局：
  - 继续沿用当前 Material 3 + Catppuccin 风格。
  - 新功能页面保持卡片式信息分组和手机端单列布局。

## 5. 数据库与数据流

本次没有新增数据库表或字段。

数据流如下：

1. 学生在 Android App 中点击底部导航进入练习、错题、我的等模块。
2. App 通过 Retrofit 调用现有后端学生接口。
3. 后端返回题库、错题、公告、用户资料、反馈结果等数据。
4. App 将数据渲染到 Fragment / Activity，并把用户操作再次提交给对应接口。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/questions` | GET | 拉取练习题库 | `page`、`pageSize`、`category` | `PageResult<Question>` |
| `/api/questions/categories` | GET | 获取练习分类 | 无 | `List<String>` |
| `/api/wrong-notebooks` | GET / POST | 获取或创建错题本 | 名称、描述 | `List<WrongNotebook>` / `WrongNotebook` |
| `/api/wrong-notebooks/{id}` | GET / PUT / DELETE | 错题本详情、修改、删除 | `id` | `NotebookDetail` / 更新结果 |
| `/api/wrong-notebooks/{id}/items` | POST | 将错题加入错题本 | `answerId` | 处理结果 |
| `/api/wrong-notebooks/{id}/items/{itemId}` | DELETE | 从错题本移除题目 | `id`、`itemId` | 处理结果 |
| `/api/results/wrong-questions` | GET | 获取全部错题 | 无 | `List<WrongQuestionGroup>` |
| `/api/users/me` | GET / PUT | 获取或更新个人资料 | 姓名、邮箱、密码字段 | `UserProfile` |
| `/api/announcements/unread` | GET | 获取未读公告 | 无 | `UnreadInfo` |
| `/api/feedbacks/my` | GET | 查询已反馈题目 ID | `questionIds` | `List<Int>` |
| `/api/feedbacks` | POST | 提交题目纠错反馈 | 题目 ID、考试 ID、反馈类型、说明 | `QuestionFeedback` |

## 9. 风险与后续优化

- 当前 App 已补齐核心学生功能，但仍以传统 View + Fragment 方式实现，后续可以继续抽离更统一的 Adapter、表单校验和公共样式组件。
- 在线练习页的文本输入监听目前以轻量方式实现，后续可继续优化 RecyclerView 复用下的输入体验。
- 错题详情页目前以题干、答案和操作为主，暂未完整呈现网页端的选项细节和导出能力，后续可继续补齐。
- 个人中心已经合并公告、主题、服务器、退出登录等能力，但如果后续模块继续增多，可以再拆分为多个二级设置页。
