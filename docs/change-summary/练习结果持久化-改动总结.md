# 练习结果持久化改动总结

## 1. 改动目标
在线练习功能原本纯前端实现，提交后不保存结果。本次将其改为调用后端 API 持久化，使学生可查看历史练习记录。

## 2. 用户视角
- **练习提交**：提交后结果保存到服务端，跳转回练习页可查看历史记录。
- **练习历史**：学生端侧边栏"练习提升"下新增"练习历史"入口，以表格展示每次练习的题目数、正确数、错误数、正确率和得分。

## 3. 前端实现
- **页面**：`StudentPractice.vue` — `submit` 调用 `api.submitPracticeApi(payload)`，不再本地判分；新增 `PracticeHistory.vue` — 展示练习历史表格。
- **路由**：`/student/practice-history`，侧边栏"练习提升"分组下新增菜单项。
- **类型**：新增 `PracticeResult`、`SubmitPracticePayload` 接口。
- **API**：`submitPracticeApi(payload)`、`getPracticeHistoryApi()`。

## 4. 后端实现
- **Entity**：新增 `PracticeResult`（`practice_result` 表映射）。
- **Mapper**：新增 `PracticeResultMapper`。
- **DTO**：新增 `SubmitPracticeRequest`（含内嵌 `PracticeAnswer`）。
- **Service**：`PracticeService.submit()` — 接收答案列表，查询题目信息进行判分，逐题插入 `student_answer`（exam_id=0），创建 `practice_result` 记录；`history()` — 按学生查询所有练习记录。
- **Controller**：`PracticeController` — `POST /api/practice/submit`、`GET /api/practice/history`。
- **SchemaInitializer**：新增 `practice_result` 表。

## 5. 数据库与数据流
1. 学生提交练习 → `POST /api/practice/submit` → 遍历答案判分 → 每条答案插入 `student_answer`（exam_id=0）→ 插入 `practice_result` → 返回 `PracticeResult`
2. 学生查看练习历史 → `GET /api/practice/history` → 查询 `practice_result WHERE student_id = ? ORDER BY submit_time DESC`

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/practice/submit` | POST | 提交练习答案 | `{ answers: [{ questionId, answer }] }` | `PracticeResult` |
| `/api/practice/history` | GET | 获取练习历史 | 无 | `PracticeResult[]` |

## 9. 风险与后续优化
- 练习答案存在 `student_answer`（exam_id=0），不与正式考试冲突。
- 后续可在练习历史页面增加"查看详情"功能，显示每道题的具体答题情况。
- 后续可增加练习的错题回顾功能。
