# 服务端草稿暂存改动总结

## 1. 改动目标
考试作答草稿原本只存于浏览器 localStorage，切换设备或清除缓存后丢失。改为服务端存储，localStorage 作为 beforeunload 兜底。

## 2. 用户视角
草稿保存逻辑不变（自动 800ms 防抖 + 页面关闭时保存），但草稿现在存储在服务端，换设备登录后仍可恢复。

## 3. 前端实现
- **StudentExamDetail.vue**：
  - `saveDraft()` 改为调用 `api.saveDraftApi()` + localStorage 兜底
  - `loadDraft()` 改为优先从服务端加载，fallback 到 localStorage
  - `clearDraft()` 改为调用 `api.clearDraftApi()` + 清除 localStorage
- **API**：新增 `saveDraftApi()`、`loadDraftApi()`、`clearDraftApi()`

## 4. 后端实现
- **表**：`draft_answer`（id, exam_id, student_id, answers TEXT, use_time, updated_at），唯一约束 (exam_id, student_id)
- **Entity**：`DraftAnswer`
- **Mapper**：`DraftAnswerMapper`
- **Controller**：`ExamController` 新增三个端点
  - `PUT /api/exams/{examId}/draft` — 保存/更新草稿
  - `GET /api/exams/{examId}/draft` — 读取草稿
  - `DELETE /api/exams/{examId}/draft` — 提交后清除草稿

## 5. 数据库与数据流
```
学生答题 → watch(800ms) → PUT /exams/{id}/draft (JSON answers + useTime)
学生刷新/重开 → mount → GET /exams/{id}/draft → 解析 answers JSON → 恢复
学生提交 → 成功 → DELETE /exams/{id}/draft
页面关闭 → beforeunload → localStorage 兜底（同步，确保不丢）
```

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/{examId}/draft` | PUT | 保存草稿 | `{ answers: [...], useTime }` | ApiResponse |
| `/api/exams/{examId}/draft` | GET | 读取草稿 | 无 | `DraftAnswer` |
| `/api/exams/{examId}/draft` | DELETE | 清除草稿 | 无 | ApiResponse |

## 9. 风险与后续优化
- beforeunload 时仍存 localStorage 作为最终兜底，确保不掉草稿
- answers 以 JSON 字符串存在 TEXT 列中，排序和查询需求简单，不另外建关联表
- 后续可给草稿加过期时间（如 7 天），定期清理未提交草稿
