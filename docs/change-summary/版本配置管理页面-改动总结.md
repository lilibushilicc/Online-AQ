# 版本配置管理页面改动总结

## 1. 改动目标

在管理后台增加「版本更新」配置页面，方便管理员在线配置 App 版本更新信息，无需直接操作数据库。

## 2. 用户视角

管理员进入「系统设置 → 版本更新」，可配置以下项：
- 版本号 (code)
- 版本名 (name)
- APK 下载链接
- 更新说明
- 是否强制更新

填写后点击「保存设置」即可生效。

## 3. 前端实现

- **路由**：新增 `/admin/config/version` 路由，复用 `AdminConfig.vue` 组件
- **菜单**：系统设置子菜单新增「版本更新」条目
- **配置项**：在 `config` 响应式对象中添加 5 个 `app.version.*` 键
- **UI**：使用 `<el-card>` + `<el-form>` 渲染版本配置表单，沿用现有保存逻辑

### 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-vue/src/router/index.ts` | 新增 `config/version` 路由 |
| `Oline-AQ-vue/src/views/admin/AdminConfig.vue` | 新增 `app.version.*` 配置项、tab 标签和配置卡片 |
| `Oline-AQ-vue/src/views/admin/AdminLayout.vue` | 系统设置菜单新增「版本更新」条目 |

## 4. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/config` | GET | 获取所有配置 | 无 | `Record<string, string>` |
| `/api/config` | PUT | 保存配置 | `{ "app.version.code": "2", ... }` | 成功消息 |

## 5. 验证方式

1. 启动前端，以管理员登录
2. 进入「系统设置 → 版本更新」
3. 填写版本信息并保存
4. 重新进入确认数据已持久化
5. 手机端确认版本检查是否生效
