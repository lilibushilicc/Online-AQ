# 学生端移动应用界面精修改动总结

## 1. 改动目标

本次改动继续针对 `Oline-AQ-app` 做学生端手机界面精修，目标是提升关键页面的信息密度、可读性和交互稳定性。

上一轮已经补齐了练习、错题本、个人中心等核心功能，这一轮重点放在体验细化：考试列表需要改为一行一个试卷；练习页需要减少 RecyclerView 输入复用带来的交互问题；错题本详情需要更清晰的层级展示；个人中心与公告卡片需要更贴近移动端信息流阅读体验。

## 2. 用户视角

- 学生在考试列表中会看到一行一个试卷，不再是双列卡片，标题、说明、状态、时间范围更容易看清。
- 学生在练习页作答时，即使未提交，也能看到当前是“未作答”还是“已填写”的提示。
- 学生切换练习题、滚动列表或回收列表项时，文本输入更稳定，不容易出现旧监听残留问题。
- 学生在错题本详情中会看到按考试分组的卡片，每道错题也有独立卡片，阅读层次更清楚。
- 学生在个人中心能看到更完整的头部信息区和更清晰的公告卡片，已读公告会弱化显示，未读公告更醒目。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 页面：
  - `ExamListFragment` 改为单列列表展示。
  - `PracticeFragment` 细化状态提示和输入监听处理。
  - `WrongBookDetailActivity` 强化分组标题、副标题和题目卡片层次。
  - `ProfileFragment` 调整个人中心头卡与公告表现。
- 组件：
  - `item_exam.xml` 新增考试说明区域，并把单卡内容扩展为更适合单列展示的样式。
  - `item_wrong_group.xml` 新增分组副标题。
  - `item_wrong_question.xml` 从普通线性布局升级为描边卡片结构。
  - `item_practice_question.xml` 的提示区域常驻显示，增强答题状态反馈。
- 交互：
  - 练习页输入框 watcher 改成可移除再绑定，避免 RecyclerView 复用带来重复监听。
  - 个人中心公告已读项使用透明度弱化，未读项保留完整强调。

## 4. 后端实现

- Controller：无改动。
- Service：无改动。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

本次仍然只涉及 Android 客户端界面与交互细化，没有修改后端接口。

## 5. 数据库与数据流

数据库无新增表或字段。

数据流保持不变：

1. App 拉取考试、练习题、错题、公告、个人资料等数据。
2. 客户端只改变展示结构和交互提示，不改变接口调用链路。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/student` | GET | 获取考试列表 | 无 | `List<Exam>` |
| `/api/questions` | GET | 获取练习题库 | `page`、`pageSize`、`category` | `PageResult<Question>` |
| `/api/results/wrong-questions` | GET | 获取全部错题 | 无 | `List<WrongQuestionGroup>` |
| `/api/wrong-notebooks/{id}` | GET | 获取错题本详情 | `id` | `NotebookDetail` |
| `/api/announcements/unread` | GET | 获取公告 | 无 | `UnreadInfo` |

## 9. 风险与后续优化

- 练习页当前已经避免了主要的重复监听问题，但如果后续题型继续增多，建议再抽离成更完整的多类型 Adapter。
- 错题详情页已明显提升层次，但目前仍以文字信息为主，后续可以继续补单选题的选项可视化展示。
- 个人中心公告已经更接近卡片流阅读体验，后续可以增加展开收起、分页或单独公告页。
