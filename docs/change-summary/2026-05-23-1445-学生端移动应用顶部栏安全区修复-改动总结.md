# 学生端移动应用顶部栏安全区修复改动总结

## 1. 改动目标

本次改动针对 `Oline-AQ-app` 学生端顶部栏做收口优化，目标是去掉过强的彩色头栏背景，同时修复标题距离屏幕顶部过近的问题。

前一轮虽然已经统一了导航与视觉风格，但顶部栏仍保留了彩色背景，和你想要的更干净布局不一致；另外启用 Edge-to-edge 后，部分页面标题会过于贴近顶部状态栏区域。本轮的核心就是把顶部栏恢复为中性色并补齐安全区。

## 2. 用户视角

- 顶部栏不再使用彩色渐变背景，整体更干净。
- 页面标题仍然保持居中，但视觉压力更小。
- 标题与状态栏之间的距离恢复正常，不再贴顶。
- 返回箭头和标题文字在浅色/暗色模式下都更稳定易读。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 布局：
  - 为主要 Fragment / Activity 根布局增加 `fitsSystemWindows="true"`。
  - 将多个页面的 `MaterialToolbar` 背景从渐变改为 `?attr/colorSurface`。
- 文字与图标：
  - 顶部栏标题改用 `?attr/colorOnSurface`。
  - 带返回按钮的页面同步设置 `navigationIconTint`，避免图标在中性色背景上对比不足。
- 覆盖页面：
  - 考试列表
  - 成绩列表
  - 练习页
  - 错题本
  - 个人中心
  - 错题详情
  - 考试详情
  - 成绩详情

## 9. 风险与后续优化

- 当前通过 `fitsSystemWindows` 已解决主要标题贴顶问题，后续如继续深入沉浸式布局，可统一抽取 WindowInsets 处理逻辑做更细粒度控制。
- 顶部栏已经回收到中性色，如果后续想保留品牌感，可以把品牌色转移到标题下方的卡片区域或操作按钮，而不是再次压回到头栏本身。
