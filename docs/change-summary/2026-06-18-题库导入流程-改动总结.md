# 题库导入流程改动总结

## 1. 改动目标

将题库导入从「上传即自动解析并入库」的一步流程，改为「解析预览 → 全选/勾选 → 一键上传」的三步流程，让管理员在上传前能预览解析结果、自由选择要导入的题目，避免错误导入。

## 2. 用户视角

1. 在「上传试题」页面选择文件、填写分类/名称、选择解析模式后，点击「解析预览」按钮。
2. 系统上传文件并解析，下方出现「解析预览」卡片，以表格形式展示所有识别到的题目。
3. 表格第一列为复选框，表头有「全选」复选框，默认所有题目为选中状态。
4. 管理员可以勾选/取消勾选任意题目，表头「全选」复选框同步显示选中或半选状态。
5. 确认选中的题目无误后，点击「一键上传 (N)」按钮，将所选题目批量入库。
6. 导入成功后，预览卡片消失，「最近导入记录」表格刷新显示新上传的文件。
7. 点击「取消」则不保存任何题目，清理已上传的临时文件。

## 3. 前端实现

- **页面**：`AdminUpload.vue`
- **组件**：`el-table` + `el-checkbox`（全选控件）、`el-table-column type="selection"`（行级复选框）
- **新增状态**：`parsedQuestions`（解析出的题目列表）、`selectedQuestions`（当前选中的题目）、`parsedFileId`（上传后文件 ID）、`previewing`（是否显示预览）、`importing`（导入加载中）
- **新增方法**：
  - `previewParse()` — 上传文件 + 预览解析（不保存到数据库）
  - `onSelectionChange()` — 监听表格行选择变化
  - `toggleSelectAll()` — 全选/取消全选
  - `confirmImport()` — 一键上传选中的题目到数据库
  - `cancelPreview()` — 取消预览并清理临时文件
- **计算属性**：`isAllSelected`、`isIndeterminate`（全选复选框状态）
- **按钮变更**：「上传并解析」改为「解析预览」

## 4. 后端实现

- **Controller**：`FileController.java`
  - `POST /api/files/{fileId}/preview` — 解析题目但不保存，返回题目列表
  - `POST /api/files/{fileId}/import` — 接收前端选中的题目列表，批量保存到数据库
- **Service**：`FileService.java`
  - `preview(fileId, category, useAi)` — 调用解析服务解析文本，返回 Question 列表（不持久化）
  - `importQuestions(fileId, questions)` — 设置 sourceFileId、清空 questionId（交由数据库自增），逐条插入，更新文件状态为 "parsed"

## 5. 数据库与数据流

```
上传文件 → 存储到本地/R2 + 写入 upload_file 表（status = "uploaded"）
      ↓
解析预览 → 读取 raw_text → 调用解析引擎 → 返回题目列表到前端（不落库）
      ↓
确认导入 → 前端提交选中的题目列表 → 批量 INSERT 到 question 表
      ↓
更新文件状态为 "parsed" → upload_file 表记录可见
```

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/files/{fileId}/preview` | POST | 解析文件但不保存，返回题目预览 | category, useAi (body JSON) | `Question[]` 题目列表 |
| `/api/files/{fileId}/import` | POST | 批量导入前端选中的题目 | `{ questions: Question[] }` | `{ fileId, questionCount }` |

## 9. 风险与后续优化

- **风险**：预览但不导入的文件会残留在服务器上（status = "uploaded"），可通过定时任务清理。
- **后续优化**：预览表格支持按题型/答案筛选排序、支持在预览阶段编辑题目、支持分页显示大量题目。
