# 学生端移动应用导航与信息表现精修改动总结

## 1. 改动目标

本次改动继续针对 `Oline-AQ-app` 的学生端 Android 界面做最后一轮导航和信息表现精修，目标是让学生端在视觉上更完整，尤其是导航、顶部栏、错题选项表现和公告空状态的一致性。

前一轮已经完成高对比品牌化配色强化，但底部导航的品牌感还不够完整，顶部栏标题也没有统一，错题详情缺少选项可视化，公告空状态也缺少一致的图标语义。本轮就集中解决这些点。

## 2. 用户视角

- 底部导航会更像一个带圆角和面板层次的品牌悬浮条，而不是普通贴底导航。
- 顶部栏标题统一居中，页面结构更稳定。
- 学生查看错题详情时，单选题和判断题能直接看到选项内容，正确答案会更明显。
- 公告为空时，不再只是简单一行字，而是有统一图标和说明文案的空状态。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 导航：
  - `activity_main.xml` 的底部导航加入新的品牌面板背景、图标尺寸和上下留白。
- 顶部栏：
  - 考试、成绩、练习、错题本、个人中心、错题详情等页面的 `MaterialToolbar` 标题统一居中。
- 错题详情：
  - `item_wrong_question.xml` 新增选项展示容器。
  - `WrongBookDetailActivity.kt` 在单选/判断题中渲染 A/B/C/D 选项，并用独立背景突出正确答案。
- 公告与空状态：
  - 新增 `ic_empty_state.xml` 和 `item_empty_state.xml`。
  - 个人中心公告为空时统一使用图标化空状态。

## 4. 后端实现

- Controller：无改动。
- Service：无改动。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

本轮仍然只修改 Android 客户端显示和交互，不涉及后端。

## 5. 数据库与数据流

数据库无改动，数据流也无变化。

客户端只是把已有错题详情数据、公告数据以更清晰的形式展示出来。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/results/wrong-questions` | GET | 获取错题 | 无 | `List<WrongQuestionGroup>` |
| `/api/wrong-notebooks/{id}` | GET | 获取错题本详情 | `id` | `NotebookDetail` |
| `/api/announcements/unread` | GET | 获取公告 | 无 | `UnreadInfo` |

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-app/app/src/main/res/layout/activity_main.xml` | 底部导航品牌悬浮条风格调整 |
| `Oline-AQ-app/app/src/main/res/drawable/bg_bottom_nav.xml` | 底部导航背景 |
| `Oline-AQ-app/app/src/main/res/layout/item_wrong_question.xml` | 错题详情新增选项区 |
| `Oline-AQ-app/app/src/main/java/com/onlineaq/student/ui/wrongbook/WrongBookDetailActivity.kt` | 选项可视化与正确答案高亮 |
| `Oline-AQ-app/app/src/main/res/layout/item_empty_state.xml` | 公告空状态统一图标样式 |
| `Oline-AQ-app/app/src/main/res/drawable/ic_empty_state.xml` | 空状态图标资源 |

## 8. 验证方式

```sh
cd Oline-AQ-app
.\gradlew.bat :app:assembleDebug
```

验证结果：Android 调试包构建成功。

## 9. 风险与后续优化

- 当前底部导航已经更有品牌感，但若后续继续精修，可以补导航选中态动画与更精致的图标系统。
- 错题详情的选项可视化目前突出的是正确答案，后续还可以把学生自己的错误选择单独标记出来。
- 空状态已经统一图标风格，后续可以继续为考试空状态、练习空状态、错题空状态设计各自专属插画。
