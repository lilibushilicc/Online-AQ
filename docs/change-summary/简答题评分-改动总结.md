# 简答题评分改动总结

## 1. 改动目标
为简答题（short_answer）增加人工评分流程：学生提交时标记为"待评分"，教师端可查看并逐题打分，评分后自动重算总分。

## 2. 用户视角
- **学生**：提交考试后，简答题显示"待评分"标签，分数为 0，待教师评分后更新。
- **教师**：侧边栏新增"评分管理"入口，列出所有待评分简答题，点击"评分"输入分数，自动更新该生的考试总分。

## 3. 前端实现
- **页面**：新增 `AdminReview.vue`，展示待评分列表，每项包含考试名称、学生姓名、题目内容、学生答案、参考答案，点击"评分"弹出输入框。
- **路由**：`/admin/review`，侧边栏"考试与试卷"分组下新增"评分管理"菜单项。
- **类型**：`ResultAnswerDetail` 新增 `reviewStatus` 字段；新增 `ReviewItem` 接口。
- **API**：`getPendingReviewsApi()`、`reviewAnswerApi(answerId, score)`。
- **详情页面**：学生端和管理端详情页对 `reviewStatus === 'pending_review'` 显示"待评分"标签。

## 4. 后端实现
- **Entity**：`StudentAnswer` 新增 `reviewStatus` 字段。
- **Controller**：`ResultController` 新增 `GET /api/results/pending-review` 和 `PUT /api/results/review/{answerId}?score=`。
- **Service**：`ResultService` 新增 `pendingReviews()` 查询所有待评分记录及关联信息；`reviewAnswer()` 更新分数及状态并调用 `recalcExamResult()` 重算总分。
- **AnswerMapHelper**：输出中增加 `reviewStatus`。
- **SchemaInitializer**：新增 `student_answer.review_status` 列，默认值 `'auto_scored'`。

## 5. 数据库与数据流
1. 学生提交 → `buildStudentAnswer()` 根据题型设置 `review_status`：简答题为 `'pending_review'`，其余为 `'auto_scored'`。
2. 教师登录 → 访问评分管理 → `GET /pending-review` 查询 `review_status = 'pending_review'` 的记录。
3. 教师打分 → `PUT /review/{answerId}?score=N` → 更新 `score`、`is_correct`、`review_status='reviewed'` → 重算该次考试的 `exam_result.total_score`、`correct_count`、`wrong_count`。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/results/pending-review` | GET | 获取待评分列表 | 无 | `ReviewItem[]` |
| `/api/results/review/{answerId}` | PUT | 评分 | `answerId`(路径), `score`(查询参数) | `ApiResponse<void>` |

## 9. 风险与后续优化
- 教师打分后前端仅刷新列表，未实时推送通知学生。
- 简答题得分可支持半对（如 5 分满分给 3 分），当前已通过 `isCorrect` = (score > 0) 间接支持。
- 后续可增加批量评分功能。
- 后续可增加评分历史记录，支持教师查看评分日志。
