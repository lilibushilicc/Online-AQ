# 并发优化第一阶段改动总结

## 1. 改动目标

消除系统在高并发场景下的 5 个已知薄弱环节：连接池未调优、`String.intern()` 锁性能风险、RateLimiter 内存泄漏、前端无请求去重、Tomcat 参数未配置。

## 2. 用户视角

无界面变化。后端请求处理能力提升，前端重复 GET 请求会被自动取消（如快速切换考试页面时）。

## 3. 前端实现

- `src/utils/request.ts`：新增 `pendingMap`（`Map<string, AbortController>`）和 `getRequestKey()`，请求拦截器中对 GET 请求执行去重——相同 method+url+params 的请求取消上一个 pending 的。响应拦截器 cleanup 对应 key。

## 4. 后端实现

- **application.yml**：新增 `spring.datasource.hikari.*`（pool=50，idle=10，timeout=5s，leak-detection=60s）和 `server.tomcat.threads.*`（max=100，accept=200）
- **ResultService.java**：`synchronized(intern)` → `LockUtils.tryLock(key, 10000)` + try/finally unlock
- **FeedbackService.java**：同上
- **EmailService.java**：同上
- **OlineAqSpringApplication.java**：加 `@EnableScheduling`
- **新增 `schedule/RateLimiterCleanupTask.java`**：`@Scheduled(fixedRate = 60s)` 调用 `RateLimiter.cleanUp()`

## 5. 数据库与数据流

无数据库变更。

## 6. 接口说明

无接口变更。

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-spring/src/main/resources/application.yml` | HikariCP + Tomcat 调参 |
| `Oline-AQ-spring/src/main/java/.../service/ResultService.java` | LockUtils 替换 synchronized |
| `Oline-AQ-spring/src/main/java/.../service/FeedbackService.java` | LockUtils 替换 synchronized |
| `Oline-AQ-spring/src/main/java/.../service/EmailService.java` | LockUtils 替换 synchronized |
| `Oline-AQ-spring/src/main/java/.../OlineAqSpringApplication.java` | 加 @EnableScheduling |
| `Oline-AQ-spring/src/main/java/.../schedule/RateLimiterCleanupTask.java` | 新增，定时清理 RateLimiter |
| `Oline-AQ-vue/src/utils/request.ts` | Axios GET 请求去重 |

## 8. 验证方式

```sh
cd Oline-AQ-spring && .\mvnw compile -q
```

Maven 编译通过，无错误。

## 9. 风险与后续优化

- HikariCP 的 `maximum-pool-size=50` 基于百人并发估算，需压测调整
- `LockUtils` 仍为 JVM 级锁，多实例部署需升级为 Redis 分布式锁（P-14）
- `RateLimiter.cleanUp()` 直接 `clear()` 所有 bucket，后续可改为只清理过期 bucket
- 前端去重仅对 GET 生效，POST/PUT/DELETE 不在去重范围内（避免幂等问题）
