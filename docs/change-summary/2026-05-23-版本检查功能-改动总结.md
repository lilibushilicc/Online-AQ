# 版本检查功能改动总结

## 1. 改动目标

为 Android 学生端增加版本检查功能：后端提供版本查询接口，Android 端在启动时自动检查，发现新版本时弹出更新对话框，支持强制更新和下载链接跳转。

## 2. 用户视角

1. 用户打开 Android App
2. App 启动时自动向服务器查询最新版本信息
3. 如果服务器版本号大于当前 App 版本号，弹出更新对话框
4. 对话框显示版本号、更新说明，用户可点击「立即更新」跳转下载链接
5. 如果 forceUpdate=true，对话框不可关闭，必须更新后才能使用

## 3. 前端实现（Android）

- **数据模型** (`data/model/AppVersionInfo.kt`)：新增 data class，包含 versionCode、versionName、downloadUrl、releaseNotes、forceUpdate 五个字段
- **接口请求** (`data/api/ApiService.kt`)：新增 `GET app/version/latest` 方法
- **启动检查** (`OnlineAQApp.kt`)：在 `onCreate` 中通过 CoroutineScope 发起异步版本查询，结果存储到 companion object 的 `latestVersion` 字段
- **展示对话框** (`ui/main/MainActivity.kt`)：在 `onCreate` 末尾调用 `checkVersionUpdate()`，对比 `BuildConfig.VERSION_CODE` 与服务器 versionCode，若需更新则弹出 AlertDialog；forceUpdate=true 时对话框不可取消

## 4. 后端实现

- **VO** (`vo/AppVersionVO.java`)：新增版本信息响应体，包含 versionCode (int)、versionName、downloadUrl、releaseNotes、forceUpdate
- **Controller** (`controller/AppVersionController.java`)：新增 `/api/app/version/latest` GET 端点，从 sys_config 表读取版本配置并返回
- **配置默认值** (`config/SchemaInitializer.java`)：新增 5 条 app.version.* 默认配置
- **路径排除** (`config/WebConfig.java`)：将 `/api/app/**` 加入 JWT 拦截器排除列表，允许未登录时查询版本

## 5. 数据库与数据流

数据流：Android App -> HTTP GET /api/app/version/latest -> AppVersionController -> ConfigService.get() -> sys_config 表 -> AppVersionVO -> JSON -> Android App

sys_config 新增键值对：

| config_key | config_value | 说明 |
|---|---|---|
| app.version.code | 1 | 最新版本号（整数递增） |
| app.version.name | 1.0 | 版本名展示 |
| app.version.download_url | (空) | APK 下载地址 |
| app.version.release_notes | (空) | 更新说明 |
| app.version.force_update | false | 是否强制更新 |

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/app/version/latest` | GET | 查询最新 App 版本 | 无 | `{code, message, data: {versionCode, versionName, downloadUrl, releaseNotes, forceUpdate}}` |

## 8. 验证方式

```sh
# 后端接口测试
curl http://localhost:8880/api/app/version/latest

# 应返回
# {"code":200,"message":"操作成功","data":{"versionCode":1,"versionName":"1.0","downloadUrl":"","releaseNotes":"","forceUpdate":false}}
```

Android 端：安装 App，修改服务器版本配置 `app.version.code = 2`，重启 App 后应弹出更新对话框。

## 9. 风险与后续优化

- 当前版本检查仅在 MainActivity 中触发，若用户未登录直接进入 LoginActivity 不会触发检查。后续可在 LoginActivity 中也添加检查逻辑。
- 下载链接为外部 URL，App 不处理 APK 下载安装逻辑，依赖系统浏览器。后续可集成下载管理器实现应用内更新。
- 暂无版本更新推送机制，需用户打开 App 后才触发检查。
