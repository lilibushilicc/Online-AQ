# UI 一致性与卡片标准化改动总结

## 1. 改动目标

修复 Android 学生端各页面之间的视觉割裂问题，统一卡片圆角、背景色、描边和阴影，消除 panel_brand/panel_info 等硬编码色值带来的不一致。

## 2. 用户视角

- 所有卡片（考试列表、成绩、错题本、练习、个人中心、公告）圆角统一为 14dp，带一致描边
- 个人中心卡片背景与其他页面一致（不再有浅色/深色跳变）
- 成绩详情汇总区也变为卡片样式，与其他卡片统一
- 考试详情计时卡片视觉上与其他卡片一致
- 页面过渡更加平滑，不再有「割裂感」

## 3. 前端实现（Android）

### 卡片圆角标准化（全部统一为 14dp）

| 文件 | 修改前 | 修改后 |
|------|--------|--------|
| `item_exam.xml` | 12dp | 14dp |
| `item_result.xml` | 12dp | 14dp |
| `item_wrong_book.xml` | 16dp | 14dp |
| `item_wrong_group.xml` | 16dp | 14dp |
| `item_wrong_question.xml` | 12dp | 14dp |
| `item_profile_announcement.xml` | 12dp | 14dp |
| `item_answer_detail.xml` | 12dp | 14dp |
| `fragment_profile.xml` 卡片1 | 18dp | 14dp |
| `fragment_profile.xml` 卡片2 | 16dp | 14dp |
| `layout_profile_settings_section.xml` | 16dp | 14dp |
| `activity_exam_detail.xml` 计时卡片 | 16dp | 14dp |
| `activity_result_detail.xml` 汇总区 | 无卡片 | 新增 14dp |

### 卡片背景标准化

- `fragment_profile.xml` 卡片1: `@color/panel_brand` → `?attr/colorSurface`
- `fragment_profile.xml` 卡片2: `@color/panel_info` → `?attr/colorSurface`
- `item_wrong_group.xml`: `@color/panel_brand` → `?attr/colorSurface`
- `activity_exam_detail.xml` 计时卡片: `@color/panel_info` → `?attr/colorSurface`
- `activity_result_detail.xml` 汇总区: `@color/panel_brand` → `?attr/colorSurface`（并改为 MaterialCardView）

### 描边和阴影标准化

- `item_result.xml`: `strokeWidth="0dp"` → `strokeWidth="1dp"` + `strokeColor="?attr/colorOutline"`
- `item_answer_detail.xml`: `strokeWidth="0dp"` → `strokeWidth="1dp"` + `strokeColor="?attr/colorOutline"`
- `item_wrong_book.xml`: 无描边 → 新增 `strokeWidth="1dp"` + `strokeColor="?attr/colorOutline"`
- `layout_profile_settings_section.xml`: 无描边 → 新增 `strokeWidth="1dp"` + `strokeColor="?attr/colorOutline"`
- 卡片 elevation 统一为 3dp

### 其他

- `activity_main.xml` fragment_container 添加 `android:background="?android:colorBackground"`
- `activity_exam_detail.xml` 添加 `app:title=""` 允许 Kotlin 动态设置标题
