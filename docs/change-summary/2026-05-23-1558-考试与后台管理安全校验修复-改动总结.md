# 考试与后台管理安全校验修复改动总结

## 1. 改动目标

本次改动聚焦两类高风险问题：

1. 学生端获取考试详情时，题目返回结构中存在把正确答案直接暴露给前端的风险。
2. 多个后台管理接口只有登录校验，没有显式管理员角色校验，学生理论上可直接调用管理接口进行越权操作。

本次目标是用最小改动把这两类风险先封住，确保学生端拿不到考试标准答案，后台操作必须由管理员发起。

## 2. 用户视角

- 学生进入考试详情页时，不会再从接口响应中拿到题目的 `correctAnswer`。
- 管理员使用题库、考试、用户、反馈、公告、文件、成绩统计等后台功能时体验不变。
- 学生如果尝试直接请求后台管理接口，将收到“仅管理员可操作”的错误提示，而不是继续执行成功。

## 3. 前端实现

- 页面：
  - 无需新增页面或组件。
- 路由：
  - 无变动。
- 状态管理：
  - 无变动。
- 接口请求：
  - 学生端仍调用原考试详情接口，但服务端返回值已脱敏。
- 样式与布局：
  - 无变动。

## 4. 后端实现

- Controller：
  - 新增 `ControllerAccess.requireAdmin()` 作为统一管理员权限守卫。
  - 为以下后台管理接口补充管理员校验：
    - `DashboardController`
    - `QuestionController`
    - `UserController`（除 `/me` 外）
    - `AnnouncementController`（创建/编辑/删除）
    - `FileController`
    - `FeedbackController`（管理端列表与处理）
    - `ResultController`（考试成绩、待评分、评分、成绩导出）
    - `ExamController`（创建、列表、编辑、发布、设置、关闭、删除、历史）
- Service：
  - `ExamService` 学生端详情返回已经通过脱敏拷贝逻辑去除 `correctAnswer`。
- Mapper：
  - 无新增。
- Entity / DTO / VO：
  - 无新增字段。
- 配置：
  - 继续沿用 JWT 拦截器写入的 `role` 请求属性，在控制器层统一判断。

## 5. 数据库与数据流

数据库无新增表字段。

数据流变化如下：

1. 登录后 JWT 拦截器把 `role` 写入 `HttpServletRequest`。
2. 后台管理接口进入控制器时先调用 `ControllerAccess.requireAdmin()`。
3. 非管理员请求会被直接拒绝。
4. 学生请求考试详情时，服务端在拼装题目列表阶段返回脱敏题目对象，不包含正确答案。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/{examId}` | GET | 获取考试详情 | `examId` | 学生端返回脱敏题目，不含正确答案 |
| `/api/questions/**` | 多种 | 题库管理 | 题目参数 | 仅管理员可调用 |
| `/api/users`、`/api/users/{id}` | 多种 | 用户管理 | 用户参数 | 仅管理员可调用，`/me` 除外 |
| `/api/files/**` | 多种 | 文件上传与解析 | 文件、分类等 | 仅管理员可调用 |
| `/api/feedbacks`（列表/处理） | GET/PUT | 反馈管理 | 状态、题目等 | 仅管理员可调用 |
| `/api/results/exam/{examId}`、`/api/results/pending-review`、`/api/results/review/{answerId}` | GET/PUT | 成绩与评分管理 | `examId`、`answerId` | 仅管理员可调用 |

## 8. 验证方式

```sh
cd Oline-AQ-spring
.\mvnw.cmd -DskipTests compile
```

验证结果：

- 后端编译通过。
- 本次未新增数据库迁移，也未改动前端页面结构。

建议手工补充验证：

1. 使用学生 token 请求 `/api/questions`、`/api/users`、`/api/exams`、`/api/results/pending-review`，确认被拒绝。
2. 使用管理员 token 请求上述接口，确认仍能正常返回。
3. 学生请求 `/api/exams/{examId}`，抓包确认题目对象中不再包含 `correctAnswer`。

## 9. 风险与后续优化

- 当前权限校验放在控制器层，已经能快速阻止越权调用；后续可继续沉到统一拦截器或 Spring Security 角色注解，减少重复入口判断。
- 学生端考试详情答案泄露已经封住，但成绩详情、导出、错题本等返回结构仍建议持续审查，避免其他接口以不同链路暴露答案。
- 当前 `UnauthorizedException` 仍返回业务错误码风格响应，后续可根据前端协议需要再统一成更明确的 HTTP 403 语义。
