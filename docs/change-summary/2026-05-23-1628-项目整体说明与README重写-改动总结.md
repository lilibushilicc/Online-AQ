# 项目整体说明与 README 重写改动总结

## 1. 改动目标

本次改动主要解决项目文档层面的两个问题：

1. 根目录 `README.md` 存在明显编码异常和信息分散问题，虽然内容很多，但整体可读性较差，不适合作为项目对外总览入口。
2. 项目当前已经同时包含 Web 管理端、Web 学生端、Android 学生端和 Spring 后端，原有说明没有把“三端 + 后端”的整体结构、模块关系和当前状态统一讲清楚。

因此本次目标是补齐一份面向全局的项目说明文档，并把 `README.md` 重写为干净、统一、可快速上手的总览版本。

## 2. 用户视角

- 新接手项目的人打开仓库后，可以直接通过 `README.md` 了解项目是什么、怎么启动、有哪些端、有哪些主要能力。
- 如果需要更系统地理解整个项目结构，可以继续查看新增的 `docs/项目整体说明.md`。
- 文档索引页也补上了新文档入口，后续查阅路径更清晰。

## 3. 前端实现

- 无前端业务代码改动。
- 本次仅更新文档，不涉及页面、组件、路由、状态管理和接口请求逻辑调整。

## 4. 后端实现

- 无后端业务代码改动。
- 本次仅新增 / 更新说明文档，不涉及 Controller、Service、Mapper、实体和数据库结构变更。

2. 检查 `docs/项目整体说明.md` 是否能覆盖项目整体定位、结构和模块关系。
3. 检查 `docs/INDEX.md` 是否已加入新文档入口。
