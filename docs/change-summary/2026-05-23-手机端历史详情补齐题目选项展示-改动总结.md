# 手机端历史详情补齐题目选项展示改动总结

## 1. 改动目标

网页端成绩历史详情已经可以查看题干、选项、你的答案和正确答案，但 Android 学生端进入历史提交详情后只显示题干与答案结果，没有把题目选项渲染出来。此次改动把手机端历史详情补齐到和网页端一致的题目展示粒度。

## 2. 用户视角

学生在 Android 端进入“成绩”后，点击某张试卷卡片可进入该试卷的历史提交列表，再点击某次提交进入详情页。

进入详情页后，现在不仅能看到每道题的题干、你的答案、正确答案和对错状态，还能看到对应的 A/B/C/D 选项内容。对于客观题，正确选项会被高亮，便于复盘。

## 3. 前端实现

- Android 页面：`ResultDetailActivity` 的答题明细列表增加选项渲染逻辑。
- Android 布局：`item_answer_detail.xml` 新增选项容器与四个选项文本区域。
- 交互表现：仅对单选题、判断题展示选项；填空题、简答题继续只显示题干和答案。
- 高亮规则：正确选项使用现有 `bg_option_correct` 样式，其余选项使用 `bg_option_neutral`。

## 4. 后端实现

- Controller：无改动，继续复用现有成绩详情接口。
- Service：无改动，接口本身已经返回 `optionA~optionD`。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

## 5. 数据库与数据流

本次改动不涉及数据库结构变更。

数据流保持不变：Android 端通过 `GET /api/results/{resultId}` 获取单次成绩详情，接口返回题目题干、选项、学生答案、正确答案和对错状态；客户端新增的只是把已有的选项字段渲染到界面上。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/results/{resultId}` | GET | 获取单次成绩详情 | `resultId` | `exam`、`result`、`answers(optionA~optionD, studentAnswer, correctAnswer, isCorrect)` |

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-app/app/src/main/java/com/onlineaq/student/ui/resultdetail/ResultDetailActivity.kt` | Android 成绩详情页补齐客观题选项渲染与正确选项高亮 |
| `Oline-AQ-app/app/src/main/res/layout/item_answer_detail.xml` | Android 成绩详情题目卡片增加选项容器 |
| `docs/考试模块.md` | 补充 Android 历史详情与网页端一致的展示说明 |
| `README.md` | 追加本次更新记录 |

## 8. 验证方式

```sh
cd Oline-AQ-app
.\gradlew.bat :app:assembleDebug
```

手工验证重点：

1. Android 学生端进入成绩列表，点击某张试卷卡片进入历史记录。
2. 点击任意一次提交进入详情页，确认客观题显示 A/B/C/D 选项。
3. 确认正确选项高亮，且你的答案、正确答案和对错状态仍正常显示。

## 9. 风险与后续优化

- 当前历史详情页对客观题使用统一的正确选项高亮，尚未额外高亮“学生实际选择”的错误选项；如需更强对比，可后续增加“我的作答”样式。
- 简答题与填空题没有选项，本次仍保持原样，只展示题干和答案。
- 如果后续要继续对齐网页端更多复盘能力，可补充分值、提交时间和题型筛选等辅助信息。
