# 学生端移动应用视觉强化改动总结

## 1. 改动目标

本次改动继续针对 `Oline-AQ-app` 的学生端界面风格做强化，目标是让当前 Android 学生端从“功能可用”提升到“品牌感更强、对比更明显、视觉层级更稳定”的状态。

用户要求采用更高对比、更有品牌感的视觉方向，因此本轮重点不再是补功能，而是统一颜色、渐变、卡片层级、提示区和重点页面版式，使考试列表、练习页、错题详情、个人中心的视觉语言更一致。

## 2. 用户视角

- 学生进入 App 后，会看到更鲜明的蓝紫到青色渐变主色。
- 顶部标题区、主按钮、头像、提示卡片的风格更统一，页面辨识度更高。
- 考试列表的一行一卡展示更有层次，说明文字、状态、分值和时间的区分更明显。
- 练习页的提示区从纯文字变成了明显的状态卡片，提交前后反馈更清楚。
- 错题详情的分组卡片和题目卡片更容易快速扫读。
- 个人中心顶部和公告列表更像完整的移动端信息流，而不是简单堆叠控件。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 配色：
  - 更新亮色与暗色 `colors.xml`，提高主色纯度和前景文字对比。
  - 新增 `gradient_center`、`panel_brand`、`panel_info`、`hint_*` 等颜色角色。
- Drawable：
  - `bg_gradient_top.xml` 和 `bg_login_btn.xml` 从双段渐变升级为三段渐变。
  - 新增 `bg_profile_avatar.xml`、`bg_hint_info.xml`、`bg_hint_success.xml`、`bg_hint_error.xml`。
- 页面与组件：
  - `item_exam.xml` 提高卡片阴影与描边，补充考试说明区。
  - `item_practice_question.xml` 增强提示区样式。
  - `item_wrong_group.xml` 使用品牌面板色块强化分组层级。
  - `item_wrong_question.xml` 使用描边卡片增强题目聚合感。
  - `fragment_profile.xml` 改造为头像式头卡 + 品牌面板色块。
  - `item_profile_announcement.xml` 提高公告卡片的层次与可读性。
- 交互：
  - 练习页根据当前状态切换不同背景提示卡。
  - 公告已读项继续弱化透明度，未读项保留完整强调。

## 4. 后端实现

- Controller：无改动。
- Service：无改动。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

本轮完全是 Android 客户端视觉强化，没有改后端接口。

## 5. 数据库与数据流

数据库无改动。

数据流无变化，依然沿用现有接口，仅改变客户端视觉表现与状态提示方式。

## 6. 接口说明

| 接口 | 方法 | 作用 | 主要参数 | 返回结果 |
|---|---|---|---|---|
| `/api/exams/student` | GET | 获取考试列表 | 无 | `List<Exam>` |
| `/api/questions` | GET | 获取练习题 | `page`、`pageSize`、`category` | `PageResult<Question>` |
| `/api/results/wrong-questions` | GET | 获取错题 | 无 | `List<WrongQuestionGroup>` |
| `/api/announcements/unread` | GET | 获取公告 | 无 | `UnreadInfo` |
| `/api/users/me` | GET | 获取个人资料 | 无 | `UserProfile` |

| `Oline-AQ-app/app/src/main/res/drawable/bg_hint_success.xml` | 练习页成功提示背景 |

## 8. 验证方式

```sh
cd Oline-AQ-app
.\gradlew.bat :app:assembleDebug
```

验证结果：Android 调试包重新构建成功。

## 9. 风险与后续优化

- 当前视觉强化已经统一到主要学生页面，但如果后续要继续增强品牌感，可以进一步补动画、微交互和状态 icon。
- 现阶段更偏“高对比功能型品牌化”，后续若要继续增强个性，可以再统一插画、空状态图和按钮动效。
