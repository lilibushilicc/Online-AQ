# 考试详情题号滚动联动改动总结

## 1. 改动目标

本次改动继续完善 `Oline-AQ-app` 考试详情页的题号导航体验，让题号导航不只是“可点击跳转”，还能够在学生手动滚动题目时自动联动当前题号高亮。

上一轮已经加入题号跳转，但如果学生不是通过点击题号，而是直接上下滑动题目列表，顶部题号高亮不会同步变化，导致导航区和实际阅读位置不一致。本次补上滚动联动逻辑，使题号导航真正成为实时定位条。

## 2. 用户视角

- 学生手动滚动题目列表时，顶部题号导航会自动切换到当前可见题目。
- 顶部题号导航会自动跟随滚动，把当前高亮题号尽量保持在可见区域中间。
- 点击题号跳转和手动滚动两种方式现在保持一致，不会出现题号高亮停在旧位置的问题。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 布局：
  - 给题号导航的 `HorizontalScrollView` 增加 `id`，便于联动滚动。
- 交互：
  - `ExamDetailActivity.kt` 中新增 `RecyclerView.OnScrollListener`。
  - 根据 `LinearLayoutManager.findFirstCompletelyVisibleItemPosition()` 和 `findFirstVisibleItemPosition()` 识别当前题号。
  - 当当前题变化时，自动调用 `highlightActiveQuestion(index)` 切换样式。
  - 高亮后同步让顶部横向题号导航滚动到当前题附近。

## 4. 后端实现

- Controller：无改动。
- Service：无改动。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

本次依旧只涉及 Android 客户端交互优化。

## 5. 数据库与数据流

数据库无改动，数据流无改动。

滚动联动仅使用客户端已有题目列表和当前位置信息，不新增任何持久化数据。

## 6. 接口说明

本次不涉及接口变化。

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-app/app/src/main/java/com/onlineaq/student/ui/examdetail/ExamDetailActivity.kt` | 增加滚动监听与联动高亮逻辑 |
| `Oline-AQ-app/app/src/main/res/layout/activity_exam_detail.xml` | 为题号导航滚动容器增加引用 id |

## 9. 风险与后续优化

- 当前“当前题”判定基于首个完全可见题或首个可见题，已经足够稳定，但后续如果希望更精确，还可以改成按屏幕中心位置匹配题目。
- 后续还可以继续扩展“跳到下一道未作答题”或“按错误标记颜色分组”的导航能力。
