# 考试管理与试卷管理去重改动总结

## 1. 改动目标

移除 AdminExams.vue 中与试卷管理（组卷、选题、编辑）重叠的功能，消除"考试管理"和"试卷管理"两个页面之间的职责混淆。组卷（创建/编辑试卷、选题、设分）统一在 AdminPapers.vue 完成，考试管理只负责发布、关闭、预览、删除和查看历史。

## 2. 用户视角

- 考试管理页面不再显示"创建考试"表单和"编辑"按钮
- 草稿状态的考试，操作栏显示"组卷"按钮，点击跳转到试卷管理页
- 已发布/已关闭的考试，操作栏显示"关闭"/"删除"按钮
- 考试卡片清晰分为：发布、关闭、预览、历史、删除

## 3. 前端代码变化

### AdminExams.vue

**删除的模板部分：**
- 创建考试 `<el-card>`（含考试名称、说明、时长、时间、重考策略、选题、已选列表等所有表单）
- 选题弹窗 `<el-dialog>`（含分类/文件/搜索筛选 + 题目列表 + 已选计数器）
- 编辑模式逻辑（`isEditing` 相关条件渲染）

**保留的模板部分：**
- 统计卡片
- 考试列表（含搜索）
- 发布弹窗（含全体/指定学生 + 乱序配置）
- 试卷预览抽屉
- 历史记录抽屉

**修改的操作按钮：**
- 草稿考试的操作：~~编辑~~ → 组卷（跳转 `/admin/papers`）
- 删除 `editExam()`、`resetForm()`、`saveExam()`、`confirmQuestionSelection()`、`openQuestionPicker()` 等不再需要的函数
- 删除 `QUESTION_TYPE_LABEL` 引用（不再需要）
- 删除 `form`、`isEditing`、`questionPickerVisible`、`pickerSearch`、`pickerCategory`、`pickerFileId`、`selectedSearch`、`selectedQuestions`、`filteredSelected`、`categoryOptions`、`uploadFiles`、`selectedScore`、`getScore`、`setScore` 等不再需要的变量/函数

**修复的问题：**
- 删除模板中重复的 `<StatCards />` 调用（之前重复了两遍）

## 4. 关键文件

## 6. 风险与后续优化

- 无风险。仅前端模板和脚本清理，不涉及后端接口变化。
