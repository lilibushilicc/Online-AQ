# 学生考试页面体验优化改动总结

## 1. 改动目标

提升学生端考试答题页面的使用体验，使其更接近传统纸质试卷的清晰结构和交互习惯。

## 2. 用户视角

### 考前概览
点击考试进入后，考前概览页面新增「试卷结构」区域，以彩色标签展示各题型题数（如「一、选择题 20 题」「二、判断题 10 题」），让考生提前了解试卷全貌。

### 题目分组显示
答题页面按题型分组：**选择题 → 判断题 → 填空题 → 简答题**，每组前有大题头卡片（蓝色主题色、左侧边框），与传统试卷排版一致。

### 导航栏分类
右侧题目导航按钮也按题型分组，每组上方显示题型标签（如「一、选择题」），按钮沿用全局题号不变。

### 题型标签
每道题右上角显示题型标签（如 `[单选]` `[判断]` `[填空]` `[简答]`），使用 `effect="plain"` 彩色标签，暗色模式下同样清晰可辨。

### 判断题圆形按钮
判断题选项从 `A. 对` / `B. 错` 改为圆形绿色「对」和红色「错」，视觉更直观。

### 当前高亮
正在查看的题目卡片增加浅蓝色背景 + 左侧蓝色边框，定位更清晰。

### 草稿恢复提示
自动恢复上次作答草稿后，页面顶部显示黄色 Alert 提示条（「已从草稿恢复，已恢复 N 道题的作答记录」），可关闭，不再仅靠一闪而过的 ElMessage。

### 导出带大题头
导出 PDF/Word/Excel 时，内容按题型分组显示大题标题。

## 3. 前端实现

- **页面**：`StudentExamDetail.vue`
- **新增数据结构**：`QuestionSection` 接口、`questionSections` 计算属性（按题型分组，固定顺序：single→judge→fill_blank→short_answer，跳过 0 题类型）
- **新增状态**：`restoredFromDraft`（标记是否从草稿恢复）
- **考前概览**：在描述列表下方插入 `v-for="section in questionSections"` 循环渲染 `el-tag`
- **题目渲染**：从扁平 `v-for="question in questions"` 改为双层 `v-for="section in questionSections"` + `v-for="(question, localIdx) in section.questions"`
- **导航栏渲染**：同样改为双层循环，每组前渲染 `.nav-section-label`
- **判断题选项**：`v-if="question.questionType === 'judge'"` 分支使用自定义 `judge-option` 样式（绿色圆/红色圆）
- **题型标签**：`el-tag effect="plain" size="large"` + 根据题型动态 `type`
- **高亮样式**：`:body-style` 增加 `background: 'var(--el-color-primary-light-9)'`
- **导入导出**：导入 `QUESTION_TYPE_LABEL` 常量

## 9. 风险与后续优化

- 导出内容使用 `questionSections` 计算属性，依赖 `questions` 数据的完整性，如 `questionType` 为未知值则不会出现在任何分组中（静默丢失）。后续建议在后端枚举类型校验。
- 判断题对/错按钮暂未做键盘快捷键（如按 A/B 键快速选择），后续可以补充。
- 导航栏分组后，当某类型题目数为 0 时不显示分组标签，连续两个题型都 0 题时可能出现空档，属于预期行为。
