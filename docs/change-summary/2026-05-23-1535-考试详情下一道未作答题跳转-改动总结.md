# 考试详情下一道未作答题跳转改动总结

## 1. 改动目标

本次改动继续完善 `Oline-AQ-app` 考试详情页导航体验，在已有题号跳转和滚动联动高亮的基础上，再增加一个更直接的辅助能力：快速跳到下一道未作答题。

学生在手机端考试时，最常见的检查动作不是随机跳题，而是快速找出“还没做的题”。如果只能靠手动滑动或逐个看题号，会比较慢。本次增加快捷按钮，专门解决漏题回查的问题。

## 2. 用户视角

- 学生在考试详情页题号导航区能看到“下一道未作答”快捷按钮。
- 点击后会优先向后查找未作答题目，并自动滚动过去。
- 如果后面没有未作答题，会回到前面继续查找。
- 如果全部题目都已作答，会弹出提示说明已经全部完成。

## 3. 前端实现

这里的“前端”指 Android App 客户端界面实现。

- 布局：
  - `activity_exam_detail.xml` 在题号导航区标题行增加 `btn_next_unanswered` 按钮。
- 交互：
  - `ExamDetailActivity.kt` 新增 `jumpToNextUnanswered()` 方法。
  - 从当前激活题号之后开始查找第一道未作答题。
  - 若后续没有，再从前面的题目中补找。
  - 找到后自动滚动并高亮对应题号；若全部已答则提示用户。

## 4. 后端实现

- Controller：无改动。
- Service：无改动。
- Mapper：无改动。
- Entity / DTO / VO：无改动。
- 配置：无改动。

本次只涉及 Android 客户端考试详情页的辅助跳题能力。

## 5. 数据库与数据流

数据库无改动，数据流无改动。

“未作答题”判断完全基于客户端当前 `answers` 状态，不新增后端数据。

## 6. 接口说明

本次不涉及接口变化。

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-app/app/src/main/res/layout/activity_exam_detail.xml` | 新增“下一道未作答题”按钮 |
| `Oline-AQ-app/app/src/main/java/com/onlineaq/student/ui/examdetail/ExamDetailActivity.kt` | 实现未作答题查找与跳转逻辑 |

## 8. 验证方式

```sh
cd Oline-AQ-app
.\gradlew.bat :app:assembleDebug
```

验证结果：Android 调试包构建成功。

## 9. 风险与后续优化

- 当前逻辑按“后面优先、前面补找”的顺序跳转，已经适合大多数考试场景。后续如果需要，还可以增加“上一道未作答题”或“只看未作答题”的筛选导航模式。
