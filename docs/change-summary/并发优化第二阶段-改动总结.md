# 并发优化第二阶段改动总结

## 1. 改动目标

引入本地缓存减少重复数据库查询，降低 DB 压力，提升只读接口响应速度。

## 2. 用户视角

无明显 UI 变化。配置页保存配置后最长 30 秒生效。公告增删改后立即生效（`@CacheEvict`）。考试页面进入更快（考试信息和题目列表命中缓存）。

## 3. 前端实现

无前端变更。

## 4. 后端实现

- **新增 `config/CacheConfig.java`** — `@EnableCaching` + `CaffeineCacheManager`，注册 `config`、`exam`、`announcements` 三个缓存区域，全局 500 条上限，TTL 30s
- **pom.xml** — 添加 `spring-boot-starter-cache` + `caffeine` 依赖
- **ConfigService** — `get()` 和 `getAll()` 加 `@Cacheable("config")`，`update()` 加 `@CacheEvict(allEntries)`
- **ExamService** — `getExam()` 加 `@Cacheable("exam")`，`listExamQuestions()` 加 `@Cacheable("exam")`，所有增删改方法加 `@CacheEvict(allEntries)`
- **AnnouncementService** — `listAll()`、`listActive()`、`getById()` 加 `@Cacheable("announcements")`，增删改加 `@CacheEvict(allEntries)`

## 5. 数据库与数据流

无数据库变更。数据流变为：`Controller → @Cacheable 检查 → (缓存命中) 直接返回 | (未命中) Service → Mapper → DB → 回填缓存 → 返回`

## 6. 接口说明

无接口变更。

## 7. 关键文件

| 文件 | 作用 |
|---|---|
| `Oline-AQ-spring/pom.xml` | 添加 cache + caffeine 依赖 |
| `Oline-AQ-spring/.../config/CacheConfig.java` | 新增，缓存管理器配置 |
| `Oline-AQ-spring/.../service/ConfigService.java` | 配置缓存 |
| `Oline-AQ-spring/.../service/ExamService.java` | 考试+题目列表缓存 |
| `Oline-AQ-spring/.../service/AnnouncementService.java` | 公告缓存 |

## 8. 验证方式

```sh
cd Oline-AQ-spring && .\mvnw compile -q
```

Maven 编译通过，无错误。

## 9. 风险与后续优化

- Caffeine 为本地缓存，多实例部署时各实例独立，写入后其他实例有最多 30s 不一致窗口
- 当前 TTL 统一 30s，后续可针对不同数据单独优化：配置类 60s、公告 120s、题目 30s
- 后续可对比压测数据确认效果
