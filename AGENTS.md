# 项目级规则

- 始终使用简体中文回复。
- 本规则仅针对当前项目 `D:\study\Systemlocation\Online-AQ` 生效。
- 根目录的 `docs/` 用于存放项目说明文档、设计说明、功能说明、接口说明、代码改动总结和使用文档。
- 根目录的 `docs/change-summary/` 用于存放每次代码改动后的总结文档。
- 根目录的 `log/` 用于存放项目日志、运行日志、构建日志和排查记录。
- 根目录的 `release/package/` 用于存放打包产物。
- 根目录的 `release/docker/` 用于存放 Docker 相关文件与镜像构建产物。
- 根目录的 `WillPlan/` 用于存放将来实施的计划、预研文件和后续规划资料。
- `Oline-AQ-vue/` 和 `Oline-AQ-spring/` 目录只保留项目源码及代码运行所需文件，不在这两个目录内新增长期沉淀文档、日志文档、规划文档和打包产物。
- 如需新增说明文档，统一放到根目录 `docs/`，不要放在 `Oline-AQ-vue/` 或 `Oline-AQ-spring/` 内。
- 如需新增代码改动总结，统一放到根目录 `docs/change-summary/`。
- 如需新增日志文件，统一放到根目录 `log/`。
- 如需新增将来实施的计划、待办方案、预研材料，统一放到根目录 `WillPlan/`。
- 每次更新项目时，同步检查根目录 `README.md`，确保项目结构、运行方式、接口说明、数据库说明和产物位置与当前代码一致。
- 每次更新项目时，检查 `docs/` 中现有说明文档是否仍适用于当前项目；如果内容过时、缺失或与当前实现不一致，必须同步更新。

## 每次代码改动后的总结文档要求

- 每次对项目代码进行新增、修改或删除后，必须生成一份 Markdown 总结文档。
- 总结文档统一存放在 `docs/change-summary/`。
- 文件命名格式：

```text
YYYY-MM-DD-HHMM-功能名称-改动总结.md
```

示例：

```text
2026-05-17-1015-login-page-implementation-summary.md
2026-05-17-1430-exam-submit-scoring-summary.md
```

- 总结文档必须从多个视角说明本次改动，不能只写“改了什么文件”。
- 总结文档至少包含以下层次：
  - 改动目标：本次为什么改。
  - 用户视角：用户能看到什么变化，操作流程有什么变化。
  - 前端视角：页面、组件、路由、状态管理、请求封装如何实现。
  - 后端视角：Controller、Service、Mapper、实体、配置如何实现。
  - 数据库视角：涉及哪些表、字段、数据流和存储变化。
  - 接口视角：涉及哪些接口，请求参数、返回数据和调用关系是什么。
  - 文件视角：新增、修改、删除了哪些关键文件，各自负责什么。
  - 运行视角：如何启动、如何验证、执行过哪些构建或测试命令。
  - 风险与后续：当前限制、潜在问题、后续可以继续优化什么。

## 示例：登录页面实现总结应覆盖的内容

- 登录页如何展示教师/学生身份选择。
- 输入账号、密码后如何调用 `POST /api/auth/login`。
- 后端如何根据 `username`、`password`、`role` 查询 `sys_user`。
- 后端如何生成 JWT Token 并返回用户信息。
- 前端如何保存 Token 和用户信息。
- 路由守卫如何根据角色跳转到教师端或学生端。
- 后续接口如何通过 `Authorization: Bearer token` 携带登录状态。
- 登录失败时如何提示错误信息。
- 涉及的文件、接口、数据库表和验证步骤。
